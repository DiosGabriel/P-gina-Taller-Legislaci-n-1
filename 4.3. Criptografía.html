Criptografía

Tradicionalmente se ha definido como la parte de la criptología que se ocupa de las técnicas, bien sea aplicadas al arte o la ciencia, que alteran las representaciones lingüísticas de mensajes, mediante técnicas de cifrado o codificado, para hacerlos ininteligibles a intrusos (lectores no autorizados) que intercepten esos mensajes.

Criptología

Disciplina científica que se dedica al estudio de la escritura secreta, es decir, estudia los mensajes que, procesados de cierta manera, se convierten en difíciles o imposibles de leer por entidades no autorizadas.

Llaves para descifrado

Asimétrica: Una fórmula matemática que utiliza dos llaves, una pública y otra privada. La llave pública es accesible para todos, mientras que la llave privada solo puede ser descifrada por su receptor.

Simétrica: Utiliza algoritmos para descifrar y encriptar documentos. Son grupos de algoritmos interrelacionados que mantienen la conexión confidencial de la información.

SHA-256

Secure Hash Algorithm 256-bit, es una función hash criptográfica que convierte texto de cualquier longitud en una cadena alfanumérica casi única de 256 bits. El resultado se conoce como valor hash o hash. La clave secreta de encriptación sha 256 se utiliza para la seguridad criptográfica. Este algoritmo hash se considera extremadamente seguro, ya que produce valores hash irreversibles y únicos.

Ejemplos

Firmas digitales: Al firmar un mensaje con una firma digital, la función hash SHA 256 crea un hash del mensaje. El contenido del mensaje puede verificarse sin revelarlo utilizando claves privadas y públicas independientes creadas durante el cifrado. Esto permite una verificación y comunicación fáciles y seguras.

Protección mediante contraseña: SHA-256 se utiliza para convertir las contraseñas de los usuarios en un valor hash antes de almacenarlas en una base de datos. Los valores hash se comparan con los introducidos por el usuario durante la verificación de inicio de sesión. Las contraseñas permanecen seguras incluso en caso de fuga de datos, ya que los valores hash SHA-256 requieren una potencia de cálculo astronómica para descifrarlos durante un ataque de fuerza bruta.

Tipos de algoritmos SHA

SHA-0: Esta fue la primera versión, lanzada en 1993, pero rápidamente se descubrieron vulnerabilidades y dejó de usarse. De hecho, sus desarrolladores, NSA, nunca revelaron la fuente de la vulnerabilidad, pero sí se retiró de inmediato el algoritmo, para darle paso a la siguiente generación SHA.

SHA-1: Introducido en 1995, se trata de un algoritmo que genera un hash de 160 bits, al igual que su predecesor el SHA-0. Aunque fue ampliamente utilizado, se ha demostrado que es vulnerable a ataques de colisión y ya no se considera seguro para aplicaciones críticas, por lo que fue retirado después del año 2010.

SHA-2: Se trata del estándar que actualmente se considera seguro para la mayoría de las aplicaciones y es el que utilizan las grandes plataformas de minerías, entre ellas Bitcoin.

SHA-3: desarrollado como una alternativa a SHA-2, el SHA-3 utiliza la función de esponja Keccak, por lo que ofrece una mayor resistencia a ciertos tipos de ataques y es adecuado para aplicaciones que requieren un nivel adicional de seguridad.
